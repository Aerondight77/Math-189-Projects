Conceptual Problems

Question 1

If we dont know how many clusters to organize our data in, we can take a brute-force approach by trying different values of K. After trying different values of K, we can use various methods to assess which value of K minimizes the within cluster variation, in Laymans terms, which describes how closely knit each cluster is. One such method is known as the Elbow method, where we graph the total within variation vs.K. We then observe the Kth value for which the curve begins to flatten telling us that after a certain K, additional clusters no longer provide us with important information.

Part D

We expected the DB Scan to give us better results due to the algorithm used to choose clusters. While the K-means algorithm attempts to cluster the data by measuring the distance of each data point to a set of central points, DB Scan clusters the data by grouping lower density areas and higher density areas separately. In this case, the density-based DB Scan method is able to better detect the distinct groupings that we can see in the data. The K-means algorithm was not able to detect this shape, yielding a less informative result.
